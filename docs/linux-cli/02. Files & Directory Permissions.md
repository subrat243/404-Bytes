# 02. Files & Directory Permissions


##### Permissions :- ######

![[permissions.jpg]]

>**chmod** = Use to change mode or permission for file and directory.

>**There are two ways :-**

1.  Symbolic mode format :-

```Copy
chmod u = rwx "test.sh"
```

**here :- 
u > Current User
g > Group
o > Other User
a > All User

**"+" > Add permission
"-"  > Remove permission
"=" > Assign permission

2.  Binary or Octal mode format :-

```Copy
chmod 444 test.sh
```
==(Each value are permission for each section (d rwx rwx rwx) from above image.)

>**For no. of permissions to assign to the user, group and others, we add the binary values for each section.**

```Copy
chmod 744 test.sh
```
==**Here** :- 7 is the sum of permission which is 4, 2 and 1 which is the permission to read, write and execute assigned to the section.

##### Directory Permission :-#####

>**Symbolic mode format :-**

```Copy
chmod u = rwx Test/
```

>**Binary mode format :-**

```Copy
chmod 744 Test/
```

```Copy
chmod -R 744 Test/
```
=="-R" is for recursive permission which change the permission of file and directory within the directory along with directory.

>**chown = Change ownership of file.**

```Copy
chown "ownname" test.sh
```

>**chgrp**  = Change group ownership'

```Copy
chgrp "grpname" test.sh
```
