## **Overview**

Linux provides commands to **monitor processes, manage system resources, and control services**. These are essential for:

- System monitoring
    
- Performance tuning
    
- Troubleshooting
    
- Cybersecurity and incident response
    

---

# Process Monitoring Commands

## top ‚Äî Real-Time Process Viewer

```bash
top
```

**Explanation**

- Displays running processes in real time
    
- Shows CPU usage, memory usage, and system load
    

### Key Information Displayed

- PID (Process ID)
    
- User
    
- CPU and memory usage
    
- Running time
    

---

## htop ‚Äî Interactive Process Viewer

```bash
htop
```

**Explanation**

- Advanced version of `top`
    
- User-friendly interface
    
- Allows scrolling, searching, and killing processes interactively
    

---

## free ‚Äî Memory Usage

```bash
free -h
```

**Explanation**

- Displays total, used, and free memory
    
- `-h` ‚Üí human-readable format (MB/GB)
    

### Example Output

```
              total   used   free   shared   buff/cache   available
Mem:           8.0G   2.5G   3.0G     200M        2.5G        5.0G
```

---

## ps ‚Äî Process Snapshot

```bash
ps aux
```

**Explanation**

- Displays a snapshot of all running processes
    

### Breakdown

- `a` ‚Üí all users
    
- `u` ‚Üí detailed format
    
- `x` ‚Üí include background processes
    

---

## kill ‚Äî Terminate Process

```bash
kill PID
```

### Example

```bash
kill 1234
```

**Explanation**

- Sends a signal to terminate a process
    
- Default signal is `SIGTERM` (graceful stop)
    

---

## Force Kill Process

```bash
kill -9 PID
```

**Explanation**

- `-9` ‚Üí SIGKILL (forcefully terminates process)
    
- Use only if normal kill fails

# Systemd ‚Äî Service Management

## systemctl ‚Äî Control Services

```bash
systemctl <action> servicename
```

---

## Common Actions

|Action|Description|
|---|---|
|start|Start service|
|stop|Stop service|
|status|Show status|
|restart|Restart service|
|reload|Reload configuration|

---

## Example

```bash
systemctl start tor
```

---

## Check Service Status

```bash
systemctl status ssh
```

---

# Services at Startup

## Check if Service is Enabled

```bash
systemctl is-enabled servicename
```

---

## Enable Service at Startup

```bash
systemctl enable servicename
```

---

## Disable Service at Startup

```bash
systemctl disable servicename
```

---

## Reload Systemd Configuration

```bash
systemctl daemon-reload
```

**Explanation**

- Reloads service configuration files
    
- Required after modifying service files
    

---

# System V ‚Äî Legacy Service Management

## service Command

```bash
service servicename <action>
```

---

## Example

```bash
service apache2 start
```

---

## Available Actions

|Action|Description|
|---|---|
|start|Start service|
|stop|Stop service|
|status|Check status|
|restart|Restart service|

---

# Important Notes

- `htop` may need installation (`sudo apt install htop`)
    
- Always verify PID before killing a process
    
- Prefer `systemctl` over `service` in modern systems
    
- Use `kill -9` only when necessary
