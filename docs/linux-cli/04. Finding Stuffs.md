# 04. Finding Stuffs

#### searching with locate :-####

>**locate** = Use to find file and directory.

```Copy
locate "filename"
```

>To find file with all exact same patterns of string :-

```Copy
locate --all "filename"
```


>We can narrow down our search by specifying a path :-

```Copy
locate /path --all "filename" 
```

>We can use extensions for search :-

```Copy
locate --all "*.txt"
```

>Here " * " is to indicate all .txt extension.

```Copy
locate --all -c "filename"
```

>-c is to prints the no. of matching string patterns

```Copy
locate --all -i "filename"
```

>-i use to override the case sensitivity

#### Finding Binaries with whereis :-####
 
>**whereis** = Use to locate a binary file. 

>This command returns not only the location of the binary but also its source and man page if they are available.

```Copy
whereis "binary filename"
```

#### Finding Binaries in the PATH Variable with which :-####

>**which** = Returns the location of the binaries in the PATH variable in Linux.

```Copy
which "binary filename"
```

#### Performing More Powerful Searches with find :- ####

>**find** = It is the most powerful and flexible of the searching utilities. It is capable of search in any designated directory and looking for a number of different parameters, including the filename but also the date of creation or modification, the owner, the group, permissions, and the size.

```Copy
find  / -type f -name "filename"
```

"/" for root directory search.
"." for current directory search.
"-type" is for file type to search, "f" for file and "d" for directory.
"-name" to specify the file name.

>The find command started at the top of the filesystem (/), went through every directory looking for the filename, and then listed all instances found.

 >A search that looks in every directory can be slow. One way to speed it up is to look only in the directory where you would expect to find the files you need. we use /paths, so we could start the search in the specific /path directory, and Linux would only search as far as its subdirectories.

```Copy
find  /etc -type f -name "filename"
```

**For case sensitive search :-**

```Copy
find  / -type f -iname "filename"
```

"-iname" use for case sensitive search.

>If the file has an extension, like filename.conf, the search will not find a match. We can remedy this  limitation by using wildcards, which enable us to match multiple characters which is " * "

```Copy
find /etc -type f -name filename.*
```

>we can use size of the file for search.

```Copy
find  / -type f -name "filename" -size +1M
```

"-size" is to specify the file size to search and for bytes we use "c", ex = 1024c in place of +1M.

>if we don't know the name but know the permission, we can use "-perm" to find the file.

```Copy
find  /etc -type f -name "filename" -perm 744
```

"-executable" is for executable file.

"!-executable" is for not executable file.

"-user" to specific user.

"-group" to specific group.

>**If we are searching in full root directory, searching as a root user is effective.**
