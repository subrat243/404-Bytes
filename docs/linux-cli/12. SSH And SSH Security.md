# 12. SSH And SSH Security

>**SSH installation:-** 

```Copy
apt install openssh-client
apt install openssh-server
```

>**Usage:-**

```Copy
ssh user@"IP ADDR"

ssh user@"IP ADDR" -p "PORT"
```

>**Example:-**

```Copy
ssh admin@192.168.0.1
```

>**Then hit enter and type password**

---
##### SSH Server Login Security

>**First thing to do is disable the root login in server, to do so:-**

```Copy
vim /etc/ssh/sshd_cofig
```

>**Then at `#Authentication:` change the `#PermitRootLogin` Yes to No. Then change the `#Maxsession` as per your need if you want.**

>**Then save and restart the service.**

```Copy
systemctl restart sshd.service
```

>==**Lock down the root-account password. Locking the root password is one layer of security, that allows you to control _who_ is using the root-account.**==

>**Remove sudo-rights from a user and lock the root password and they cannot access the root-account without using exploits.**

```Copy
passwd -l root
```

---

**Generating client ssh-key:-**

```Copy
ssh-keygen -t rsa

```
>**Hit enter, after that enter the passphrase and then save.**

>**Then save the generated ssh-key to the server.**

```Copy
ssh-copy-id admin@192.168.0.1
```

>**Hit enter and then enter the password.**

>**Then disable password authentication in server.**

```Copy
vim /etc/ssh/sshd_config
```

>**Then change `#PasswordAuthentication` Yes to No. Then restart the service.**

>**To login with other Computer/Desktop we will use the private key (`id_rsa`).**

```Copy
ssh -i id_rsa admin@192.168.0.1
```

>==`#Note = The private key need to be saved in other computer to login`==

>**Now we can login using passphrase of the generated key.**

>**Change the permission of the private key `id_rsa` to protect.**

```Copy
chmod 400 id_rsa
```

##### Copying file from client to server using scp

>**scp = Secure copy (remote file copy program).**

```Copy
scp "filename" admin@192.168.0.1:/path
```

##### SSH Brute Force Protection With Fail2Ban

>**Set the fail2ban for startup:**
```Copy
systenctl enable fail2ban.service
```

>**Configuring the fail2ban config file, to do so move to fail2ban directory:**
```Copy
ls -al /etc/fail2ban/
```

>**Looking into `jail.conf` file:**
```Copy
vim jail.conf
```

>**Instead of configuring `jail.conf`, create new `jail.local` file to configure:**
```Copy
nano jail.local
```

>**CODE:-**
```Copy
[sshd]
enable = true/false
port = ssh
filter = sshd
logpath = /var/log/auth.log
maxretry = 3                //Change value accordingly
bantime = 3600              //Time in seconds
ignoreip = <IP address>
```
