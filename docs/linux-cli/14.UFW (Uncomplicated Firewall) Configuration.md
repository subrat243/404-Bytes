>**UFW, or Uncomplicated Firewall, is an interface to `iptables` that is geared towards simplifying the process of configuring a firewall. While `iptables` is a solid and flexible tool, it can be difficult for beginners to learn how to use it to properly configure a firewall. Developed to ease iptables firewall configuration, `ufw` provides a user friendly way to create an IPv4 or IPv6 host-based firewall. By default UFW is disabled.**

>**Installation**
```Copy
sudo apt-get install ufw
```

#### Enable/Disable And Status

>**To Enable the ufw service and setup for startup:**
```Copy
sudo ufw enable
```

>**To Disable the ufw service and setup for startup:
```Copy
sudo ufw disable
```

>**Check the Status of ufw:
```Copy
sudo ufw status
```

>**Check the Status with verbose, it will show more info in Status:
```Copy
sudo ufw status verbose
```

>**Check the Numbered Status:
```Copy
sudo ufw status numbered
```

---
#### Default Policies of ufw  

>**By default this command will allow all incoming connections:
```Copy
sudo ufw default allow incoming
```

>**By default this command will deny all incoming connections:
```Copy
sudo ufw default deny incoming
```

>**By default this command will allow all outcoming connections:
```Copy
sudo ufw default allow outcoming
```

>**By default this command will deny all outcoming connections:
```Copy
sudo ufw default deny outcoming
```

#### Allow/Deny The Rules And Services

>**It will show the present apps and services on system 
```Copy
sudo ufw app list
```

>**To Allow the particular Service:
```Copy
sudo ufw allow <service name>
```

>**Example:
```Copy
sudo ufw allow ssh
```

>**To Deny the particular Service:
```Copy
sudo ufw deny <Service Name>
```

>**Example:
```Copy
sudo ufw deny ssh
```

>**To Allow/Deny incoming TCP and UDP connection from a specific Port:
```Copy
sudo ufw allow/deny <Port>/<Protocol>
```

>Here the protocol is optional, Port number only can set the rules for both TCP and UDP.

##### Allow:-
**Example**
>To allow the incoming connection on port 80:
```Copy
sudo ufw allow 80
```

>To allow the incoming TCP connection on port 80:
```Copy
sudo ufw allow 80/tcp
```

>To allow the incoming UDP connection on port 80:
```Copy
sudo ufw allow 80/udp
```

##### Deny:-
**Example**
>To deny the incoming connection on port 80:
```Copy
sudo ufw deny 80
```

>To deny the incoming TCP connection on port 80:
```Copy
sudo ufw deny 80/tcp
```

>To deny the incoming UDP connection on port 80:
```Copy
sudo ufw deny 80/udp
```

#### Advance Syntax

>We can use a full syntax, specifying the source and destination addresses, ports and protocols.

>**To allow the connection from a particular IP address:
```Copy
sudo ufw allow from <IP address>
```
>Example:-
```Copy
sudo ufw allow from 192.168.0.0
```

>**To allow the connection from a subnet:
```Copy
sudo ufw allow from <subnet>
```
>Example:-
```Copy
sudo ufw allow from 192.168.0.0/24
```

>**To allow the connection by specifying Port and IP address:
```Copy
sudo ufw allow from <Target> to <Destination> port <Port Number>
```
>Example:-
```Copy
sudo ufw allow from 192.168.0.0 to any port 22
```

>**To allow the connection by specifying Port, IP address and Protocol:
```Copy
sudo ufw allow from <Target> to <Destination> port <PortNumber> proto <Protocol>
```
>Example:-
```Copy
sudo ufw allow from 192.168.0.0 to any port 22 proto tcp
```

#### Deleting and Resetting

>**To Reset the firewall to it's default:
```Copy
sudo ufw reset
```
>Note: Make sure not to use when deleting particular rule.

>**To Delete the particular rules from firewall:
```Copy
sudo ufw delete <RuleNumber>
```
