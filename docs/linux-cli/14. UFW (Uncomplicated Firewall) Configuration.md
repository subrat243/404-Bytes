## **Overview**

UFW (Uncomplicated Firewall) is a user-friendly interface for managing firewall rules using `iptables`. It simplifies firewall configuration and is commonly used for:

- Securing servers
    
- Controlling network traffic
    
- Restricting access to services
    
- Basic cybersecurity defense
    

By default, UFW is **disabled**.

---

# Installation

```bash
sudo apt-get install ufw
```

**Explanation**  
Installs the UFW firewall package on the system.

---

# Enable / Disable and Status

## Enable UFW

```bash
sudo ufw enable
```

**Explanation**

- Activates firewall
    
- Applies rules
    
- Enables at startup
    

---

## Disable UFW

```bash
sudo ufw disable
```

**Explanation**  
Stops firewall and disables it at startup.

---

## Check Status

```bash
sudo ufw status
```

---

## Verbose Status

```bash
sudo ufw status verbose
```

**Explanation**  
Displays detailed information including default policies.

---

## Numbered Rules

```bash
sudo ufw status numbered
```

**Explanation**  
Shows rules with numbers for easy deletion.

---

# Default Policies

## Allow All Incoming Connections

```bash
sudo ufw default allow incoming
```

---

## Deny All Incoming Connections (Recommended)

```bash
sudo ufw default deny incoming
```

---

## Allow All Outgoing Connections

```bash
sudo ufw default allow outgoing
```

---

## Deny All Outgoing Connections

```bash
sudo ufw default deny outgoing
```

---

## Best Practice

```bash
sudo ufw default deny incoming
sudo ufw default allow outgoing
```

**Explanation**  
Blocks unauthorized incoming traffic while allowing outbound connections.

---

# Allow and Deny Rules

## List Available Applications

```bash
sudo ufw app list
```

---

## Allow a Service

```bash
sudo ufw allow ssh
```

**Explanation**  
Allows SSH connections.

---

## Deny a Service

```bash
sudo ufw deny ssh
```

---

# Port-Based Rules

## Allow Port

```bash
sudo ufw allow 80
```

---

## Allow TCP Port

```bash
sudo ufw allow 80/tcp
```

---

## Allow UDP Port

```bash
sudo ufw allow 80/udp
```

---

## Deny Port

```bash
sudo ufw deny 80
```

---

## Deny TCP/UDP Ports

```bash
sudo ufw deny 80/tcp
sudo ufw deny 80/udp
```

---

# Advanced Rules

## Allow from Specific IP

```bash
sudo ufw allow from 192.168.0.1
```

---

## Allow from Subnet

```bash
sudo ufw allow from 192.168.0.0/24
```

---

## Allow Specific IP to Specific Port

```bash
sudo ufw allow from 192.168.0.1 to any port 22
```

---

## Allow with Protocol

```bash
sudo ufw allow from 192.168.0.1 to any port 22 proto tcp
```

---

## Deny Rules

Same syntax applies using `deny` instead of `allow`.

---

# Deleting and Resetting Rules

## Delete Rule by Number

```bash
sudo ufw delete 1
```

**Explanation**  
Deletes rule number 1 from the list.

---

## Reset Firewall

```bash
sudo ufw reset
```

**Explanation**

- Removes all rules
    
- Resets to default state
    
- Use carefully
    

---

# Practical Examples

## Secure Server Setup

```bash
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow ssh
sudo ufw enable
```

---

## Allow Web Traffic

```bash
sudo ufw allow 80
sudo ufw allow 443
```

---

## Restrict SSH to Specific IP

```bash
sudo ufw allow from 192.168.1.10 to any port 22
```

---

# Important Notes

- Always allow SSH before enabling UFW to avoid lockout
    
- Use numbered rules for easy management
    
- Prefer restrictive rules (deny by default)
    
- Test rules before deploying in production
